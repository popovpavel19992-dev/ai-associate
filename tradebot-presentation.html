00TL;DR для занятых

Одна минута чтения — чтобы сразу понять, стоит ли копать глубже.

Что делает бот

  • Автоматически выбирает top-10 пар на Bybit по ликвидности / ATR / ADX.
  • На 15m-свечах строит композитный сигнал из 7 индикаторов (RSI, MACD, EMA-cross, StochRSI, OBV, Ichimoku, Volume).
  • Фильтрует сигналы через сентимент (F&G + CoinGecko), MTF-подтверждение (1h/4h/1d), торговую сессию и корреляцию портфеля.
  • Вход — лимитным ордером с TTL 5 мин; SL/TP — на основе ATR; есть trailing-stop и DCA-доливка.
  • Каждые 24 ч ML-pipeline тюнит пороги стратегии на своих закрытых сделках с walk-forward-валидацией и авто-rollback.
  • HMM-детектор определяет режим (trend/flat/volatile) и подгружает свой набор порогов.

02Генерация сигналов

Композитный score по 7 индикаторам на 15m-таймфрейме.

Индикаторы и их вклад

ИндикаторПараметрыВклад в score
RSI14+1 если <30 · −1 если >70
MACD12/26/9+1 если line>signal & hist>0
−1 обратно
EMA Cross9/21+1 если fast>slow · −1 обратно
Stoch RSI14, smooth 3/3+1 если %K<20 · −1 если >80
OBV TrendEMA(20) от OBV+1 bullish · −1 bearish
Ichimoku9/26/52+1 если tenkan>kijun & price>cloud · −1 обратно
VolumeMA(20)±1 (усиление направления)
Диапазон score: −12 … +12. Это фактически «голосование» индикаторов с усилением объёмом.

Решение LONG / SHORT / WAIT

# signal_generator.py
SIGNAL_THRESHOLD = 4   # поднят с 3 после прод-метрик
                          # — убирает ~64% убыточных сетапов

if score >= +4:  return LONG
if score <= −4:  return SHORT
return WAIT
Порог LONG: ≥ +4 Порог SHORT: ≤ −4 Сохраняется signal.strength для MTF-bypass и ML
Обрати внимание: порог подняли с 3 до 4 уже после прод-данных — это эмпирическая корректировка, а не академическое обоснование. Это один из пунктов, на который стоит посмотреть критично.

03Отбор торговых пар

Пересканирование каждые 60 минут, всегда работаем с top-10.

Формула рейтинга пары

pair_score = 0.40·volume_score
           + 0.35·atr_score
           + 0.25·adx_score
  • volume_score = vol₂₄ʜ / max(vol₂₄ʜ в батче)
  • atr_score = (atr/price) нормированный к максимуму в батче
  • adx_score = min(ADX / 50, 1.0) — силa тренда

Пайплайн

  1. Фильтр по объёму ≥ $5M за 24 ч
  2. Нормализация символов (убираем :USDT)
  3. Фетч 30 свечей 15m для каждого кандидата
  4. Расчёт трёх скор-компонент
  5. Сортировка и top 10
  6. Ре-скан каждые 60 мин
  7. Fallback: кэш → хардкод BTC, ETH, SOL, XRP, DOGE, ADA, AVAX, LINK

04Риск-менеджмент

Статический сайзинг + ATR-based SL/TP + многоуровневые гейты.

Position sizing

notional = balance × max_position_pct   # 5%
qty       = notional / price
min_notional = $5.0                  # иначе skip
max_position_pct: 5% max_open_positions: 5 leverage: 10x daily_loss_limit: 3%
С плечом 10x и сайзингом 5% от депо, нотионал одной позиции = ~50% баланса, но маржа = 5%. Все 5 позиций одновременно = 25% маржи заблокировано, нотионал 2.5× депозита.

SL / TP (ATR-based)

SL_ATR_MULT = 5.0
TP_ATR_MULT = 10.0
MIN_SL_PCT  = 1%           # floor

LONG:
  SL = max(entry − 5·ATR, entry·0.01)
  TP = entry + 10·ATR

SHORT:
  SL = entry + 5·ATR
  TP = max(entry − 10·ATR, entry·0.01)

# fallback если ATR недоступен:
SL = ±2% · TP = ±4%
Соотношение R/R ≈ 1:2 по мультипликаторам. На практике R/R зависит от заполнения по лимитке и проскальзывания.

Trailing stop

TRAIL_ATR_MULT = 0.5
trail_step = 0.5 · ATR

LONG:  new_sl = price − trail_step
        if new_sl > cur_sl: update
SHORT: new_sl = price + trail_step
        if new_sl < cur_sl: update

Стоп двигается только в сторону прибыли. Никогда не затягивается против нас.

Gate: can_open_trade()

Новая сделка блокируется, если хотя бы один пункт истинен:

  • Уже открыто ≥ 5 позиций
  • Дневной PnL ≤ −balance × 3%
  • Drawdown от пика ≤ −20% → полный halt
  • Серия из ≥ 5 убытков подряд → пауза 60 мин
  • Кулдаун после SL/TTL на этой паре (30 мин)
  • ATR/price < 0.15% (слишком тихо)

05Сентимент-фильтр

Глобальный фон + по конкретной монете. Пороги зависят от режима рынка.

Источники

  • Fear & Greed Index (alternative.me) — 0..100, кеш 24 ч
  • CoinGecko per-coin — bullish% − bearish%, диапазон −1..+1, кеш 60 мин
  • Поддержка 20 тикеров (BTC, ETH, SOL, BNB, XRP, ADA, DOGE, AVAX, DOT, MATIC, LINK, UNI, ATOM, LTC, FIL, NEAR, APT, ARB, OP, SUI)
Если монета не в списке — используется медиана сентимента по недавним сделкам. Если API недоступен — сделки пропускаются (safe fail-open).

Пороги блокировки по режимам

РежимF&G block LONGF&G block SHORTCoin block LONGCoin block SHORT
trend< 15> 85< −0.4> +0.4
flat< 25> 75< −0.6> +0.6
volatile< 10> 90< −0.3> +0.3

Логика: LONG не берём в «экстремальном страхе», SHORT не берём в «экстремальной жадности». На плоском рынке пороги шире (меньше сделок).

06Режимы рынка — HMM-детектор

Gaussian Hidden Markov Model с 3 скрытыми состояниями.

Обучение

  • Фичи: log-returns и rolling-std(14) по log-returns
  • Окно обучения: 252 свечи 15m (~17 дней)
  • Минимум данных: 50 свечей
  • Переобучение — 1 раз в сутки

Семантический мэппинг

  • trend — состояние с наибольшим средним return
  • flat — с наименьшей дисперсией
  • volatile — с наибольшей дисперсией

Применение параметров

# trader/engine.py
detect regime каждый цикл
candidate regime per pair
if regime стабилен >= 24 часа:
    switch active params → preset(regime)

Подгружается пресет из config.yaml: пороги RSI, F&G, coin-sentiment для данного режима. Это мягкий переключатель — без резких прыжков.

Подтверждение 24 ч создаёт лаг: когда рынок уже сменил режим, бот продолжает торговать старым пресетом почти сутки. Trade-off: стабильность vs отзывчивость.

07Multi-timeframe gate

Подтверждение на старших таймфреймах перед входом.

for tf in ["1h", "4h", "1d"]:
    fetch 30 candles
    compute EMA(9), EMA(21)

    if signal == LONG  and ema_fast < ema_slow: BLOCK
    if signal == SHORT and ema_fast > ema_slow: BLOCK

    # bypass дневного фильтра если сигнал очень сильный:
    if tf == "1d" and signal.strength >= 5:
        continue
Идея: не торговать против 1h/4h EMA-тренда. Дневной таймфрейм можно «проколоть» очень сильным сигналом (score ≥ 5) — это компромисс, чтобы не пропускать мощные развороты.

09Защитный слой

Drawdown halt + correlation filter + alerts.

Drawdown halt

  • Пиковый баланс трекается каждый цикл
  • DD = (cur − peak) / peak
  • Порог halt: −20% → все новые входы блокируются
  • Alert на −15% (cooldown 60 мин)

Correlation filter

  • Матрица Pearson на returns
  • 720 свечей 1h (~30 дней)
  • Обновление каждый час
  • Блок если |corr| > 0.80 с любой открытой позицией
Не даёт «дублировать» риск: открыли BTC-long → ETH-long скорее всего будет заблокирован.

Alerts (fire-once + cooldown)

ПравилоПорогCD

pnl_usdt−$150 / день120 м

10Исполнение: лимитные ордера + DCA

Вход только лимитным ордером с TTL, доливка — опционально.

Limit order entry

LONG:  limit = bid × (1 − 0.0005)
SHORT: limit = ask × (1 + 0.0005)

TTL = 300 sec
poll every 5 sec
on fill:      обновить TradeRow.entry
on expire:    cancel, exit_reason="ttl_expired",
              cooldown 15 мин на паре
if fill < 10% qty: "micro_fill_rejected"
Все отменённые / частично исполненные ордера исключаются из W/L и дневного PnL — чтобы статистика не портилась «псевдо-сделками».

DCA (усреднение)

dca_multipliers: [1.0, 2.0, 3.5]  # в ATR
max_dca_exposure_pct: 0.20
single_order_max_pct: 0.05
enabled: false   # по умолчанию выкл

Пример: BTC @ $40k, ATR $200 →

  • Level 0: просадка $200 (1× ATR) → доливка
  • Level 1: просадка $400 → следующая
  • Level 2: просадка $700 → последняя

После каждой доливки пересчитываются средняя цена входа и новые SL/TP от неё. Общий экспозиш ≤ 20% баланса.

DCA по умолчанию выключен. На ревью стоит обсудить — нужна ли эта опция в принципе или это путь к катастрофе на сильных трендах.

11ML-пайплайн: авто-тюнинг стратегии

Каждые 24 часа пересматриваем пороги на своих закрытых сделках.

Feature set (14)

  • RSI
  • MACD hist
  • EMA trend (±1/0)
  • Volume above avg
  • Signal strength
  • F&G value
  • Coin sentiment
  • Hour UTC
  • Day of week
  • Side (BUY/SELL)
  • Leverage
  • StochRSI-K
  • OBV trend
  • Ichimoku signal

Target: is_win (0/1).

Модель и селекция

  1. SelectKBest (F-score) — перебор K=3..N, выбираем по CV-accuracy. Folds: min(5, max(2, n/5)).
  2. GridSearchCV по Random Forest — 81 комбинация: n_estimators∈{50,100,200}, max_depth∈{3,5,8}, min_samples_split∈{2,5,10}, min_samples_leaf∈{1,2,4}, n_jobs=2.
  3. Получаем feature importances.
  4. Минимум данных: 20 сделок (меньше → uniform importances).

Strategy optimizer

FEATURE_PARAM_MAP = {
  "rsi":                 [(rsi_oversold, +1), (rsi_overbought, −1)],
  "fear_greed_value":    [(fg_block_long, +1), (fg_block_short, −1)],
  "coin_sentiment_score": [(coin_block_long, −1), (coin_block_short, +1)],
}

max_step = baseline × 10% × importance
if WR < 50%: сужаем полосы (меньше сделок)
else:            расширяем (больше сделок)

bounds: baseline ± 30%

Пороги RSI обязательно сохраняют порядок oversold < overbought — иначе сброс к baseline.

Walk-forward + rollback

  • TimeSeriesSplit валидация — уважаем временной порядок.
  • Если новые параметры хуже текущих на hold-out → reject.
  • Иначе — применяем и пишем новую версию в strategy_params.
  • Каждые 10 сделок проверка эффективности:
    WR_at_apply − WR_recent ≥ 10% → rollback.
  • Rollback cascade: лучшая историческая версия с WR ≥ 40% → иначе baseline.
Это не «ML-альфа-модель». Это online hyperparameter tuner, который подгоняет пороги классических индикаторов под недавнее поведение пары + своего трейдстатистики.
