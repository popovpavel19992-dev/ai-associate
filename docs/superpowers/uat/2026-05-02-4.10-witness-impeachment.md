## Backend UAT script

```

```

Covers:
1. `attachStatement` for 2 case documents → 2 junction rows persisted, 0cr
2. `runScanFlow` happy path → scan row inserted with contradictions + claims, +4cr
3. Same args → cache hit, 0cr, same row id
4. Regenerate via `regenerateSalt` → new row inserted, +4cr, both rows persist
5. Detach one statement → re-scan = cache miss + new row, +4cr (statements_hash changes)
6. Detach all statements → `NoStatementsError`, 0cr
7. Non-beta org → `NotBetaOrgError`, 0cr
8. Cleanup deletes witness list (cascades witness → statements → scans)

## Notes

- Cache key = `witnessId:statementsHash:caseStateHash:regenerateSalt`. Detaching/attaching statements changes statementsHash and forces a cache miss on next call.
- `case_witness_impeachment_scans.statements_snapshot` stores `{statementId, documentId, kind, date, contentHash}` so the UI can detect staleness even after statements drift.
- Two-step Claude pipeline: per-statement `extractClaims` → `scanContradictions` against Voyage-RAG'd evidence (statement docs excluded from RAG to avoid self-matches).
- Reuses opposing-counsel posture intel (4.7) via `opposingCounselPostures` (latest row per case) when present to bias contradiction prioritization.
- 4cr per scan is refunded on Claude/DB error (preserved through `try/catch` around `decrementCredits`).
- Local UI UAT may be blocked by an upstream voyageai package bug — recommend production smoke after merge if local dev throws on Voyage import.
- Pre-existing tsc errors in `scripts/uat-4.6-demand-letter-ai.ts`, `scripts/uat-4.7-opposing-counsel.ts`, `scripts/uat-4.8-settlement-coach.ts`, `scripts/uat-4.9-deposition-branches.ts`, `src/server/services/stripe.ts` are unrelated to 4.10.
