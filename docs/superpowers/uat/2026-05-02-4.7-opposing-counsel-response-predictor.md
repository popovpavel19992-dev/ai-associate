## Manual scenarios (run after migrations)

### Identity & posture (Opposing Counsel tab)
- [ ] Open a case in the test org → "Opposing Counsel" tab is visible.
- [ ] Click "+ Add attorney" → name + firm + bar# + state → submit. Row appears in list.
- [ ] Click "Posture" → readout renders within ~10s. Confirm 2 credits charged in ledger.
- [ ] Re-open posture → instant return, no credit charge (cache hit).
- [ ] Non-beta org → tab content shows "private beta" message; no buttons visible.

### Filing prediction (inline buttons)
- [ ] Motion drafter → "Predict opposing response" button appears next to other actions.
- [ ] Click → scorecard renders with low/high ranges + per-field confidence dots.
- [ ] 2 credits charged. Click Regenerate → fresh prediction, +2 credits.
- [ ] Repeat for Demand Letter and Discovery Response drafters.
- [ ] Discovery: confirm Q/A pairing in the AI context is correct (sparse drafts no longer misalign — fixed in 7d5f1f6).

### Auto-extract (Inngest)
- [ ] Upload an opposing-authored motion PDF → Inngest fires `opposing-counsel/extract-signature`.
- [ ] After extraction, `documents.suggested_attorney_json` populated for that doc.
- [ ] Suggested-attorney banner appears in Opposing Counsel tab → "Add" creates case_party + attaches profile.
- [ ] Image uploads do NOT trigger the extract step (cost guard).

### Beta gate / errors
- [ ] Non-beta org: direct tRPC call to `predictResponse` → FORBIDDEN.
- [ ] No opposing counsel attached: dialog shows "Add attorney to this case", no credit charged.
- [ ] Multiple opposing counsel: dialog shows dropdown, no credit until selected.
- [ ] Force Claude error (invalid `ANTHROPIC_API_KEY`) → error toast + 2cr refunded in ledger.

### Accessibility
- [ ] Dialogs announce as modal (role=dialog + aria-modal).
- [ ] Close (✕) buttons have aria-label and disable while predict is pending.
- [ ] Confidence dots have screen-reader labels.
