# 4.8 — Settlement Negotiation Coach (design)

**Date:** 2026-05-02
**Phase:** 4.8
**Status:** Spec — awaiting user review
**Predecessor patterns:** 4.2 Strategy, 4.3 Motion Drafter, 4.6 Demand Letter, 4.7 Opposing Counsel Predictor (`src/server/services/{case-strategy,motion-drafter,demand-letter-ai,opposing-counsel}/`)
**Migration:** 0062
**Beta gate:** existing `STRATEGY_BETA_ORG_IDS`

## Goal

Plaintiff-side AI coach for settlement negotiation. Two products in one module:

1. **BATNA / ZOPA readout** — strategic snapshot. AI extracts damage components from case docs, lawyer can override, server computes BATNA range and ZOPA from formula, sensitivity breakdown shows BATNA at different win-probability points.
2. **Counter-offer recommender** — when defendant sends an offer in the existing `case_settlement_offers` track, AI proposes 3 variants (aggressive / standard / conciliatory) each with rationale + risk readout. Server bounds-checks each counter to `[BATNA, last_demand]`.

Both surfaces live inline in the existing Settlement tab — no new page.

### Database (migration 0062)

```sql
-- BATNA / ZOPA snapshot for a case
CREATE TABLE settlement_coach_batnas (
  id                    uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id                uuid NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  case_id               uuid NOT NULL REFERENCES cases(id) ON DELETE CASCADE,
  cache_hash            text NOT NULL,

  -- AI-extracted damage components (cents)
  damages_low_cents     bigint,
  damages_likely_cents  bigint,
  damages_high_cents    bigint,
  damages_components    jsonb NOT NULL,

  -- AI-estimated win probability range
  win_prob_low          numeric(3,2),
  win_prob_likely       numeric(3,2),
  win_prob_high         numeric(3,2),

  -- AI-estimated remaining costs + time
  costs_remaining_cents bigint,
  time_to_trial_months  integer,
  discount_rate_annual  numeric(4,2),

  -- Server-computed
  batna_low_cents       bigint NOT NULL,
  batna_likely_cents    bigint NOT NULL,
  batna_high_cents      bigint NOT NULL,
  zopa_low_cents        bigint,
  zopa_high_cents       bigint,
  zopa_exists           boolean NOT NULL,

  sensitivity_json      jsonb NOT NULL,

  reasoning_md          text NOT NULL,
  sources_json          jsonb NOT NULL,
  confidence_overall    text,
  has_manual_override   boolean NOT NULL DEFAULT false,

  created_at            timestamptz NOT NULL DEFAULT now(),

  CONSTRAINT scb_confidence_check
    CHECK (confidence_overall IS NULL OR confidence_overall IN ('low','med','high')),
  CONSTRAINT scb_batna_low_high_check CHECK (batna_low_cents <= batna_high_cents),
  CONSTRAINT scb_damages_low_high_check CHECK (
    damages_low_cents IS NULL OR damages_high_cents IS NULL
      OR damages_low_cents <= damages_high_cents
  ),
  CONSTRAINT scb_zopa_low_high_check CHECK (
    zopa_low_cents IS NULL OR zopa_high_cents IS NULL
      OR zopa_low_cents <= zopa_high_cents
  ),
  CONSTRAINT scb_winprob_range_check CHECK (
    (win_prob_low IS NULL OR win_prob_low BETWEEN 0 AND 1)
    AND (win_prob_likely IS NULL OR win_prob_likely BETWEEN 0 AND 1)
    AND (win_prob_high IS NULL OR win_prob_high BETWEEN 0 AND 1)
  )
);
CREATE UNIQUE INDEX scb_cache_uq
  ON settlement_coach_batnas(org_id, cache_hash)
  WHERE cache_hash IS NOT NULL;
CREATE INDEX scb_case_idx
  ON settlement_coach_batnas(case_id, created_at DESC);

-- Counter-offer recommendations (per incoming defendant offer)
CREATE TABLE settlement_coach_counters (
  id                    uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id                uuid NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  case_id               uuid NOT NULL REFERENCES cases(id) ON DELETE CASCADE,
  offer_id              uuid NOT NULL REFERENCES case_settlement_offers(id) ON DELETE CASCADE,
  batna_id              uuid REFERENCES settlement_coach_batnas(id) ON DELETE SET NULL,
  cache_hash            text NOT NULL,

  variants_json         jsonb NOT NULL,
  -- shape: [
  --   { tag: 'aggressive',   counter_cents, rationale_md, risk_md, confidence, clamped: bool },
  --   { tag: 'standard',     ... },
  --   { tag: 'conciliatory', ... }
  -- ]

  bounds_low_cents      bigint NOT NULL,
  bounds_high_cents     bigint NOT NULL,
  any_clamped           boolean NOT NULL DEFAULT false,

  reasoning_md          text NOT NULL,
  sources_json          jsonb NOT NULL,
  confidence_overall    text,

  created_at            timestamptz NOT NULL DEFAULT now(),

  CONSTRAINT scc_confidence_check
    CHECK (confidence_overall IS NULL OR confidence_overall IN ('low','med','high')),
  CONSTRAINT scc_bounds_check CHECK (bounds_low_cents <= bounds_high_cents)
);
CREATE UNIQUE INDEX scc_cache_uq
  ON settlement_coach_counters(org_id, cache_hash)
  WHERE cache_hash IS NOT NULL;
CREATE INDEX scc_offer_idx
  ON settlement_coach_counters(offer_id, created_at DESC);
```

Notes:
- BATNA = immutable cache row. Override damages = new row with `has_manual_override=true` and override-hash mixed into `cache_hash` (audit history preserved).
- Counter `batna_id ON DELETE SET NULL` — counter survives BATNA delete as audit trail (Phase 4 pattern).
- `variants_json` as single jsonb (always 3 variants — rows-per-variant overkill).
- `bounds_low/high` snapshotted on the counter row so a re-read shows what BATNA + last demand were applied at recommendation time.

### Cache hash composition

```
batna.cache_hash    = sha256(case_id + case_state_hash + override_hash + regen_salt)
case_state_hash     = sha256(case_id + max(documents.created_at) + max(case_settlement_offers.offered_at))
override_hash       = sha256(JSON.stringify(overrides ?? null))

counter.cache_hash  = sha256(case_id + offer_id + batna_id + offer.amount_cents +
                             last_demand_cents + regen_salt)
```

7-day TTL via `created_at` predicate at read time (no background sweep).

### Compute (compute.ts)

Pure server functions. No Claude calls.

```
batna_X = win_prob_X × damages_X − costs_remaining − time_discount(time_to_trial_months, discount_rate_annual)

defendant_BATNA_estimate (heuristic):
  if posture exists with settle_high → damages_likely × posture.settle_high
  else                                → damages_likely × 0.7

zopa_exists = batna_likely <= defendant_BATNA_estimate
zopa_low    = batna_likely
zopa_high   = defendant_BATNA_estimate
(when zopa_exists=false, both null)

sensitivity table:
  for win_prob in [0.30, 0.45, 0.60, 0.75]:
    row = { winProb, batnaLow: f(damages_low),
            batnaLikely: f(damages_likely),
            batnaHigh: f(damages_high) }

counter bounds:
  bounds_low  = batna.batna_likely
  bounds_high = last_demand.amount_cents (fallback: damages_likely if no prior demand)
  for each variant: if counter_cents < bounds_low → clamp to bounds_low, mark clamped
                    if counter_cents > bounds_high → clamp to bounds_high, mark clamped
```

## Data flow

### Flow A — computeBatna

1. tRPC `settlementCoach.computeBatna({ caseId, overrides? })`
2. Compute `case_state_hash`, `override_hash`, `cache_hash`
3. Cache lookup → hit returns row, no charge
4. `decrementCredits(userId, 3)` → if false → `InsufficientCreditsError`
5. `sources.collectDamagesSources(caseId)` — Voyage RAG
6. `extract.run({ sources, caseSummary })` → Claude sonnet structured JSON
7. Apply overrides field-by-field over extracted (manual > AI)
8. Optionally load latest `opposing_counsel_postures` for caseId (defendant-BATNA heuristic)
9. `compute.batnaZopa(...)` — pure
10. `INSERT settlement_coach_batnas`, return row
11. Catch wrapping steps 5–10: `refundCredits(userId, 3)` and re-throw

### Flow B — recommendCounter

1. tRPC `settlementCoach.recommendCounter({ caseId, offerId })`
2. Load latest BATNA for case → if none → `{ needsBatna: true }`, no charge
3. Load offer with `id=offerId, org=orgId, response='pending', from_party='defendant'` → if missing → `NotFoundError`, no charge
4. Compute last demand: `MAX(amount_cents) WHERE case_id AND from_party='plaintiff' AND offer_type IN ('opening_demand','counter_offer')`
5. Compute `cache_hash`; cache lookup → hit returns row, no charge
6. `decrementCredits(userId, 2)`
7. Build context: BATNA, offer, lastDemand, posture (optional), recentOffers, demandLetter (optional)
8. `recommend.run(ctx)` → Claude sonnet 3 variants
9. Apply bounds clamp → mark `any_clamped` as needed
10. `INSERT settlement_coach_counters`, return row
11. Catch wrapping 7–10: refund 2cr and re-throw

### Flow C — overrideComponents

Wrapper around Flow A with overrides. Result row has `has_manual_override=true`.

## UI surface

### BatnaZopaCard (top of Settlement tab)

- Header: confidence dot
- BATNA range: `$low – $high (likely $likely)`
- ZOPA section: range or "no settlement zone" message when `zopa_exists=false`
- Damages components list: each row labelled with low/likely/high + source attribution
- Sensitivity panel: table of BATNA at win_prob ∈ {30, 45, 60, 75}%
- "Reasoning" + "Sources" expanders
- Computed-at timestamp + Recompute (3cr) button
- Edit-components button → opens `<ComponentsEditor />`

When no BATNA exists: empty-state CTA "Compute BATNA / ZOPA (3 credits)".

### CounterRecommenderButton (per offer row)

Renders only when offer is `from_party='defendant'` AND `response='pending'` AND beta-enabled.
Click → `<CounterDialog />` modal with three variant cards stacked. Each card shows:
- Tag chip: aggressive / standard / conciliatory
- Counter amount in cents → display formatted USD
- Confidence dots
- Rationale paragraph
- Risk paragraph
- "Use this →" button: prefills the existing 3.4 "New counter offer" dialog with `amount_cents` set
- Per-variant ⚠ badge if `clamped=true`

Below variants: shared reasoning + sources expanders. Regenerate (2cr) button.

### ComponentsEditor

Modal listing each damage component row (label + low/likely/high cents) editable. Add/remove rows. On submit → `overrideComponents` mutation.

### Beta gate

UI hidden client-side via `trpc.opposingCounsel.isBetaEnabled` (reused — same env var). Server enforced via `assertBetaOrg` in orchestrator throwing `NotBetaOrgError` → tRPC FORBIDDEN.

## Error handling

| Path | Behaviour |
|---|---|
| No case_documents | extract returns sparse JSON, `confidence_overall='low'`, sensitivity rendered with wide ranges |
| Voyage returns 0 chunks | Proceed with `caseSummary` only, downgrade confidence to `low` |
| Claude error / refusal | Refund credits, error toast, no row inserted |
| Claude invalid JSON / Zod fail | Refund + Sentry log |
| Counter requested without BATNA | Short-circuit `needsBatna: true`, no charge |
| Counter for non-pending or non-defendant offer | NotFoundError, no charge |
| Last demand missing | Use `damages_likely` as `bounds_high` fallback |
| `batna_likely > last_demand` | Bounds collapse → use `[batna_low, batna_high]`, flag in `any_clamped` |
| `zopa_exists=false` | Counter still computed; UI shows "no settlement zone" warning above variants |
| Variant exceeds bounds | Clamp + per-variant `clamped=true` + `any_clamped=true` |
| Manual override low > high | Zod refusal at tRPC, no charge |
| Cache hash collision | UNIQUE catches, return existing row |
| Beta-gate bypass via direct tRPC | TRPCError FORBIDDEN |
| Concurrent computeBatna same hash | Second hits new row via UNIQUE → returns it |
| `damages_likely_cents = 0` | BATNA = `−costs_remaining` (negative); flag `confidence='low'`, UI shows "insufficient damages signal" |

**Refund rule** (Phase 4 pattern): charge → LLM → on throw, `refundCredits(userId, COST)` in catch, re-throw.
