# 4.10 ‚Äî Witness Statement Cross-Check (impeachment) (design)

**Date:** 2026-05-02
**Phase:** 4.10
**Status:** Spec ‚Äî awaiting user review
**Predecessor patterns:** 4.5 Discovery Response, 4.7 Opposing Counsel Predictor, 4.8 Settlement Coach, 4.9 Deposition Branches
**Migration:** 0064
**Beta gate:** existing `STRATEGY_BETA_ORG_IDS`

## Goal

AI cross-checks a single witness's prior statements (deposition transcripts, declarations, RFA responses, etc.) for **self-inconsistencies** within their own statements AND **factual contradictions** with the rest of the case docs. Output: pair-of-quotes contradictions with severity classification + 2-3 ready-to-use cross-examination questions per contradiction.

This is impeachment-grade output ‚Äî the most actionable AI product for a trial lawyer.

## Decisions (from brainstorm)

| # | Question | Choice |
|---|----------|--------|
| 1 | Statement source | **A** ‚Äî lawyer manually attaches docs to witness as statements (auto-detect deferred to 4.10c) |
| 2 | Comparison kinds | **D** ‚Äî self-inconsistencies (within one witness) + statement vs case docs. Cross-witness conflicts deferred to 4.10b |
| 3 | Output granularity | **C** ‚Äî pair quotes + severity (direct/inferred/tangential) + suggested impeachment questions |
| 4 | Trigger | **C** ‚Äî per-witness scan + opt-in re-scan when statements added |
| 5 | Pricing | **A** ‚Äî flat 4cr per witness scan, 7-day cache |

| 7 | UI | **A** ‚Äî inline section in witness detail view |
| 8 | Scope guard | v1 strict ‚Äî see Out-of-Scope below |

## Architecture

### File layout

```

  scan.ts              ‚Äî Claude sonnet ‚Äî find contradictions across claims + case docs +
                          emit suggested impeachment questions
  orchestrator.ts      ‚Äî cache ‚Üí credits ‚Üí load statements ‚Üí extract (parallel) ‚Üí
                          sources ‚Üí scan ‚Üí tx insert
  index.ts             ‚Äî re-exports

src/server/trpc/routers/witness-impeachment.ts            (new)
  procedures: attachStatement, detachStatement, listStatementsForWitness,
              runScan, getScan

src/server/db/schema/case-witness-statements.ts            (new ‚Äî junction)
src/server/db/schema/case-witness-impeachment-scans.ts     (new ‚Äî cache rows)

src/components/cases/trial-prep/
  witness-impeachment-section.tsx     (new ‚Äî wrapper in witness detail)
  attach-statements-dialog.tsx         (new ‚Äî multi-select case_documents)
  contradictions-list.tsx              (new ‚Äî renders results)

```

Re-uses: `claude.ts`, `credits.ts`, Voyage from `case-strategy/voyage`, `documents`, `case_witnesses`. Inherits `z.preprocess` confidence-alias normalization (4.8), staleness snapshot pattern (4.9), defense-in-depth orgId scoping (4.9 lesson).

### Two-step Claude pipeline

`extract.run` per statement ‚Üí claims_json. Then `scan.run` over compact claims + sources ‚Üí contradictions. Reasoning: a single deposition can be 50k+ tokens; one-shot scan over multiple statements blows context. Atomic claims with locators are compact and let the scan step cite back into source statements precisely.

### Database (migration 0064)

```sql
-- Lawyer-attached statements: which documents are statements by which witness.
CREATE TABLE case_witness_statements (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id          uuid NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  case_id         uuid NOT NULL REFERENCES cases(id) ON DELETE CASCADE,
  witness_id      uuid NOT NULL REFERENCES case_witnesses(id) ON DELETE CASCADE,
  document_id     uuid NOT NULL REFERENCES documents(id) ON DELETE CASCADE,

  statement_kind  text NOT NULL,
  statement_date  date,
  notes           text,

  attached_by     uuid NOT NULL REFERENCES users(id),
  created_at      timestamptz NOT NULL DEFAULT now(),

  CONSTRAINT cws_kind_check CHECK (
    statement_kind IN ('deposition','declaration','affidavit','rfa_response',
                       'rog_response','prior_testimony','recorded_statement','other')
  )
);
CREATE UNIQUE INDEX cws_witness_doc_uq
  ON case_witness_statements(witness_id, document_id);
CREATE INDEX cws_witness_idx
  ON case_witness_statements(witness_id, created_at);
CREATE INDEX cws_case_idx
  ON case_witness_statements(case_id);

-- Per-witness immutable scan cache.
CREATE TABLE case_witness_impeachment_scans (
  id                  uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id              uuid NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  case_id             uuid NOT NULL REFERENCES cases(id) ON DELETE CASCADE,
  witness_id          uuid NOT NULL REFERENCES case_witnesses(id) ON DELETE CASCADE,
  cache_hash          text NOT NULL,

  statements_snapshot jsonb NOT NULL,
  -- shape: [{ statementId, documentId, statementKind, statementDate, contentHash }]

  claims_json         jsonb NOT NULL,
  -- shape: [{ statementId, claims: [{ id, text, locator, topic }] }]

  contradictions_json jsonb NOT NULL,
  -- shape: [
  --   { id, kind: 'self'|'evidence', severity: 'direct'|'inferred'|'tangential',
  --     summary, leftQuote: { text, statementId|documentId, locator },
  --     rightQuote: { text, statementId|documentId, locator },
  --     impeachmentQuestions: string[] }   -- 2-3 questions
  -- ]

  reasoning_md        text NOT NULL,
  sources_json        jsonb NOT NULL,
  confidence_overall  text,

  created_at          timestamptz NOT NULL DEFAULT now(),

  CONSTRAINT cwis_confidence_check
    CHECK (confidence_overall IS NULL OR confidence_overall IN ('low','med','high'))
);
CREATE UNIQUE INDEX cwis_cache_uq
  ON case_witness_impeachment_scans(org_id, cache_hash)
  WHERE cache_hash IS NOT NULL;
CREATE INDEX cwis_witness_idx
  ON case_witness_impeachment_scans(witness_id, created_at DESC);
CREATE INDEX cwis_case_idx
  ON case_witness_impeachment_scans(case_id, created_at DESC);
```

Notes:
- `(witness_id, document_id) UNIQUE` ‚Äî same doc can't be attached twice; can attach to multiple witnesses (joint declaration).
- `statements_snapshot[].contentHash` lets UI detect stale scans even if the doc was re-uploaded with new text.
- `claims_json` persisted as intermediate step output ‚Äî debug aid + future re-scan optimization (deferred).
- `kind: 'self' | 'evidence'` ‚Äî `self` = within-witness contradiction, `evidence` = statement vs case doc.
- `leftQuote`/`rightQuote` reference EITHER `statementId` OR `documentId`, not both ‚Äî UI renders accordingly.
- Detaching a statement does NOT cascade scan rows ‚Äî they're audit trail. UI shows stale badge via snapshot diff.

### Cache hash composition

```
case_state_hash    = sha256(case_id + max(documents.created_at WHERE org_id = ?))
statements_hash    = sha256(sorted [statementId:sha256(extractedText)])
cache_hash         = sha256(witness_id + statements_hash + case_state_hash + regen_salt)
```

7-day TTL via `created_at` predicate at read time.

## Compute / extraction conventions

- Claude sonnet model literal `"claude-sonnet-4-6"`
- Inline `getAnthropic()` + `messages.create` (canonical: `demand-letter-ai/classify.ts`)
- DI via optional `deps.anthropic`
- Errors propagate; orchestrator catches and refunds
- Zod with `z.preprocess` on confidence/severity enums to tolerate `medium ‚Üí med`
- Zod refines: contradiction `impeachmentQuestions` length ‚àà [2, 3]; quote requires either `statementId` or `documentId`, not both

## Data flow

### Flow A ‚Äî attach/detach/list (free CRUD)

`attachStatement({ caseId, witnessId, documentId, statementKind, statementDate?, notes? })`
- Verify witness belongs to caseId + org via outer join
- Verify document belongs to caseId + org
- INSERT into junction (UNIQUE prevents dup)
- Return row

`detachStatement({ caseId, statementId })`
- DELETE WHERE id=statementId AND org_id=ctx.orgId

`listStatementsForWitness({ caseId, witnessId })`
- SELECT junction with documents JOIN
- Return [{ id, documentId, filename, statementKind, statementDate, notes, attachedAt, status (extracted/pending) }]

### Flow B ‚Äî runScan

1. tRPC `witnessImpeachment.runScan({ caseId, witnessId, regenerate? })`
2. `assertBetaOrg(orgId)` + verify witness in case+org
3. Load attached statements with their `documents.extractedText`
   - 0 statements ‚Üí `NoStatementsError`, no charge
   - any doc with empty `extractedText` ‚Üí `NotExtractedError`, no charge
4. Compute `statements_hash`, `case_state_hash`, `cache_hash`
5. Cache lookup ‚Üí hit returns row, no charge
6. `decrementCredits(userId, 4)` ‚Üí false ‚Üí `InsufficientCreditsError`
7. Try (refund-wrapped):
   1. `extract.run` per statement (parallel, max concurrency 3) ‚Üí claims per statement
   2. If all statements yield 0 claims ‚Üí `NoClaimsError`, refund
   3. `sources.collectEvidenceSources({ caseId, witnessName, excludeDocumentIds, query })` ‚Äî Voyage top-K=15, filters out statement docs
   4. Load latest `opposing_counsel_postures` for caseId (best-effort, null on miss)
   5. `scan.run({ witness, statements, claims, sources, posture, caseSummary })` ‚Üí contradictions
   6. INSERT scan row with statements_snapshot + claims_json + contradictions_json + reasoning + sources
8. Catch wrapping 7.1‚Äì7.6: `refundCredits(userId, 4)`, re-throw

### Flow C ‚Äî getScan (read)

```
SELECT row WHERE witness_id=? AND org_id=? AND case_id=?
ORDER BY created_at DESC LIMIT 1
```
Returns row or null.

UI computes staleness: compare `row.statements_snapshot[].contentHash` to current attached-statements list (after recomputing each contentHash from current `documents.extractedText`).

## UI surface

### Mount point

Inside existing trial-prep witness detail view, add `<WitnessImpeachmentSection witnessId={...} caseId={...} />` as a new section beneath any existing witness fields.

### Section states

**No statements attached:**
```
[+ Attach statement]
"Attach witness statements to enable AI cross-check"
```

**Statements attached, no scan:**
```
Statements (3): ‚Ä¢ Smith depo ‚Ä¶  ‚Ä¢ Smith decl ‚Ä¶  ‚Ä¢ RFA ‚Ä¶
[Run cross-check (4 credits)]
```

**Scan exists:**
```
confidence: HIGH ¬∑ Last scan: 30 min ago ¬∑ 5 contradictions
[+ Attach] [Re-scan (4cr)]
‚öÝ Statements changed since last scan   (only if stale)

<contradictions-list />
```

### Contradiction card

Per item:
- Severity badge with color (direct=red, inferred=amber, tangential=zinc) and dot pattern
- Kind chip (`self` | `evidence`)
- Summary line
- A: quote + locator + source filename
- B: quote + locator + source filename
- "Suggested impeachment questions" expandable list (2-3 items)

`tangential` items collapsed by default; expandable via summary count badge.

### Components

- `witness-impeachment-section.tsx` ‚Äî owns the section; loads `listStatementsForWitness` + `getScan`; computes staleness; beta-gated.
- `attach-statements-dialog.tsx` ‚Äî multi-select from extracted case_documents (filter `status='ready'`). Per row: doc picker + statementKind dropdown + optional date + notes. Submit creates N junction rows.
- `contradictions-list.tsx` ‚Äî renders `contradictions_json`. Branches A/B quote rendering by `statementId` vs `documentId`.

### Beta gate

Section returns `null` when not beta (re-uses `trpc.opposingCounsel.isBetaEnabled`). Server `assertBetaOrg` enforces FORBIDDEN on direct tRPC calls.

## Error handling

| Path | Behaviour |
|---|---|
| 0 statements attached | Short-circuit `NoStatementsError`, no charge |
| Any attached doc has empty extractedText | `NotExtractedError`, no charge ‚Äî UI prompts to wait |
| Voyage 0 chunks | Proceed, downgrade `confidence_overall='low'` |
| Voyage finite/dim guard fail | Empty sources, log warn |
| `VOYAGE_API_KEY` missing | Empty sources |
| Posture missing | Proceed without |
| extract.run throws on any statement | Refund 4cr, error toast |
| extract.run invalid JSON / Zod fail | Same |
| extract yields 0 claims across all statements | `NoClaimsError`, refund |
| scan.run Claude error / refusal | Refund + Sentry + retry toast |
| scan.run invalid JSON / Zod fail | Same |
| scan unknown `kind` or `severity` | Zod rejects ‚Üí refund |
| scan `medium` for severity | `z.preprocess` normalizes |
| scan returns quote with neither statementId nor documentId | Zod refine rejects ‚Üí refund |
| Attach: doc not in same case | tRPC NOT_FOUND, no charge |
| Attach: doc not extracted yet | Allow attach; scan refuses until extracted |
| Detach during scan in progress | Junction delete safe; resulting row may be stale-flagged immediately |
| Cache hash collision | UNIQUE catches ‚Üí return existing row |
| Concurrent scan same hash | Second hits new row via UNIQUE ‚Üí returns it |
| Witness deleted | Cascade drops statements + scans (FK CASCADE) |
| Beta-gate bypass | TRPCError FORBIDDEN |

**Refund rule** (Phase 4): charge ‚Üí extract + sources + scan + insert in `try` ‚Üí catch refunds + re-throws.
