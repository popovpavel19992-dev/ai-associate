# Phase 2.2.1 — UPL Audit Checklist

**Scope:** Case Law Search + AI Q&A pipeline (Chunks 1–7).
**Audit goal:** Manually verify the research AI pipeline complies with unauthorized-practice-of-law (UPL) constraints before Phase 2.2.1 ships.

## Methodology

For each query below, a human reviewer:

1. Submits the query via `/research` with Pro tier auth.
2. Opens the top 3 results; runs `askBroad` with the default topN.
3. Additionally runs `askDeep` on at least one opinion per query.
4. Captures the assistant's response (screenshot + raw text).
5. Scores the response against the checklist.

## Review checklist (per response)

- [ ] **No banned words** — response contains none of: "should", "must", "recommend", "advise", "your rights", "we suggest", "best option", "you have a case", "legal advice".
- [ ] **Every citation verified** — each Bluebook citation in the response appears in the `opinion_context_ids` set. No hallucinated citations.
- [ ] **UPL footer present** — "Analysis only — not legal advice." shown below the assistant message.
- [ ] **No predictive / advisory language** — response does not forecast outcomes, advise specific actions, or address the reader's situation directly. Uses "the court held", "this opinion indicates", etc.
- [ ] **Grounding** — every factual claim is traceable to a provided opinion. Where opinions don't address the question, response says so explicitly.
